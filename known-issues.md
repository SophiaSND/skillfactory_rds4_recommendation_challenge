## Низкая производительность

Поскольку база данных (sqlite3) интегрирована прямо в docker-контенер, и файл базы данных (src/flask/data/flask.db) "весит" 541MB, то запущенный контейнер требователен к размеру оперативной памяти

Бесплатный тариф на heroku позволяет расчитывать только на машину с оперативной памятью размером 512MB, чего явно недостаточно, и это очень заметно сказываетcя на производительности

Поэтому для знакомства с production-сервисом рекомендуется локальный запуск docker-контейнера. Поскольку production-сервис предназанен для демонстрации прототипа, использующего модель рекомендательной системы, а не для реального использования в production, низкая производительность на сервере не является критичным недочетом

## Не работает кнопка `Clear filters`

На оригинальном сайте `amazon.com` кнопка `Clear filters` сбрасывает все ранее установленные параметры фильрации/сортировки.

В нашем случае оставлена лишь визуальная заглушка для имитации интерфейса оригинального сайте

Для демонстрации прототипа недочет некритичный

## Разваливается макет страницы с отзывами на десктопе

На некоторых страницах (например, https://evening-badlands-35661.herokuapp.com/item/5902?is_review=1&start_review=20) макет странциы выглядит неаккуратно из-за большой длины текста отзывов.

Для демонстрации прототипа недочет некритичный
